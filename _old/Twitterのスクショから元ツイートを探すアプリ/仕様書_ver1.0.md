個人開発や小規模チームでの開発を想定した、実用性重視の「要件定義書（仕様書）」 を作成しました。
このままNotionやGitHubのREADMEに貼り付けて開発をスタートできる構成にしています。
アプリ名（仮）：TweetSource Finder
1. プロダクトの概要
Twitter（X）のスクリーンショット画像から、文字情報を解析（OCR）し、Google検索を経由してオリジナルの投稿URLを特定するツール。
 * コンセプト: 「あのスクショ、元ネタどこ？」を3秒で解決する。
 * ターゲット: SNSヘビーユーザー、まとめサイト利用者、ファクトチェックを行いたい層。
 * 提供価値: 検索の手間削減、フェイク画像の判別支援。
2. 機能要件（MVP: Minimum Viable Product）
まずはリリースするために必要な「最小限の機能」です。
A. メイン機能
 * 画像アップロード:
   * カメラロールからの選択。
   * OS共有（Share Extension）からの受け取り: 他のアプリ（インスタや写真アプリ）の「共有」ボタンから直接このアプリに画像を渡せる機能（※ここがUXの肝）。
 * OCR解析（文字認識）:
   * 画像内のテキストを抽出する。
   * 日本語・英語に対応。
 * 自動検索実行:
   * 抽出したテキストを整形し、ブラウザ（またはアプリ内ブラウザ）で検索結果を表示する。
 * 履歴機能:
   * 過去に検索した画像の履歴リスト（ローカル保存）。
B. 検索ロジック（重要）
XのAPIを使わず、Google Custom Search API（または通常のWeb検索）を活用してコストを下げる。
 * クエリ生成ルール:
   * 基本形: site:x.com "抽出した本文の一部"
   * 除外ワード: 画像に含まれがちな「いいね」「リポスト」「時間」「プロモーション」などのノイズ単語を削除してから検索にかける。
3. UI/UX フロー（画面設計）
画面1: ホーム（待機画面）
 * シンプルに大きな「画像をアップロード」ボタンを配置。
 * 下部に「最近の履歴」リストを表示。
画面2: 解析・確認画面（省略可能だが推奨）
 * OCRで読み取ったテキストを編集できる画面。
   * 理由: OCRが誤読した場合（例: l と 1 の間違い）、ユーザーが手動で修正して再検索できるようにするため。
 * 「このテキストで検索」ボタン。
画面3: 結果表示
 * Google検索結果の一覧を表示。
 * ユーザーがタップすると、Xアプリまたはブラウザへ遷移。
4. 技術スタック案（推奨）
個人開発での「作りやすさ」と「コスト」を重視した構成です。
| 項目 | 推奨技術 | 理由 |
|---|---|---|
| アプリ開発 | Flutter or React Native | iOS/Androidを同時に作れる。共有機能の実装もしやすい。 |
| OCRエンジン | Google ML Kit (On-device) | 無料かつ高速。サーバー通信不要でプライバシー的にも安心。 |
| 検索エンジン | Google Programmable Search Engine | 無料枠あり（1日100回）。超えると有料だが、X APIより遥かに安い。 |
| サーバー | Firebase (Auth/Firestore) | 履歴保存や認証が必要な場合。MVPならローカル保存のみでサーバーレスでも可。 |
5. 開発のロードマップ
 * フェーズ 1（プロトタイプ）:
   * 画像をアップして、OCRで文字が出るだけのアプリを作る（検索はまだ）。
   * ここでOCRの精度（特に日本語の長文）を確認する。
 * フェーズ 2（検索連携）:
   * 抽出テキストをGoogle検索URLに変換してブラウザを開く機能を実装。
   * https://www.google.com/search?q=site:x.com+... を叩くだけならAPI不要で完全無料。
 * フェーズ 3（共有機能）:
   * iOS/Androidの「共有メニュー」からの起動を実装。ここが出来ると一気にアプリらしくなる。
 * フェーズ 4（リリース）:
   * ストア審査提出。
6. 懸念点と対策（リスク管理）
 * OCR精度:
   * 課題: スクショに映り込んだ「電池残量」や「広告」まで読み取ってしまう。
   * 対策: テキスト抽出後、正規表現で「%」「:（時刻）」などのパターンを除去するフィルタ処理を入れる。
 * Xの仕様変更:
   * 課題: site:x.com で検索しても、X側が検索除け（noindex）を強化するとヒットしなくなる可能性がある。
   * 対策: その場合は「Yahoo!リアルタイム検索」など、別のインデックスを持つ検索エンジンへの切り替えオプションを用意する。
