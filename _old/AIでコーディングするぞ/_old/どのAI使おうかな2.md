第1章：アーキテクチャ選定の思考プロセス
なぜ「月額制アプリ（SaaS）」ではなく「API組み合わせ型」を選定したのか、その論理的な帰結を詳述する。
1.1 課題の構造化
ユーザーの要望には、互いにトレードオフ（両立困難）となる要素が含まれていた。
 * 「最高性能のAIを使いたい」 → 通常、高額なコスト（月額$20〜$40）がかかる。
 * 「PCとスマホの両方で使いたい」 → PC特化ツール（Cursor）はスマホが弱く、スマホ特化（ChatGPT）はPCのローカル編集ができない。
 * 「予算は月3,000円」 → 複数の有料サービス（例：Cursor $20 ＋ ChatGPT Plus $20）を契約すると予算オーバー（$40）になる。
1.2 候補となるパターンの詳細検証と棄却理由
【案1】Cursor Pro 単独契約案 ($20/月)
 * 検証: PCでのコーディング体験は現時点(2024-2025)で最高峰。
 * 致命的欠陥: Cursorにはスマホアプリが存在しない。
 * 結果: 外出先での「思考整理」を行うには、スマホで別の手段が必要になる。そこで無料版ChatGPTを使うと「最高性能ではない」という要件に反し、有料版を追加すると「予算超過」となる。よって不採用。
【案2】ChatGPT Plus / Gemini Advanced 単独契約案 ($20/月)
 * 検証: スマホアプリは優秀。最新モデル（GPT-4o等）が使える。
 * 致命的欠陥: これらはブラウザ/アプリ完結型であり、ユーザーのPC内にある「ローカルファイル（.csファイルなど）」を直接開き、編集し、保存する権限を持たない。コードをコピペして貼り付ける作業が発生し、「ローカルドキュメントを直接編集してほしい」という要件を満たせない。よって不採用。
【案3】VS Code + Cline + API従量課金案 (推奨案)
 * 検証:
   * PC: VS Code（無料）に拡張機能「Cline」を入れることで、API経由でCursorと同等のファイル編集機能を持たせることができる。
   * スマホ: API対応のチャットアプリ（Chatbox等）を使えば、PCと同じAPIキーを使って会話できる。
   * コスト: 「PCとスマホで一つの財布（APIキー）を共有する」形になるため、無駄な二重課金が発生しない。
 * 結論: 予算3,000円以内で「PCでのファイル編集」と「スマホでの最高性能チャット」を両立できる唯一の物理的構成であるため、これを採用。
第2章：AIモデル（脳みそ）選定の思考プロセス
API利用を前提とした場合、どのAIモデルを採用すべきか。直近1ヶ月（2024年12月）の技術動向とベンチマークに基づき詳細比較を行う。
2.1 OpenAI "GPT-4o" の評価と棄却理由
 * 性能: 非常に高いが、直近のAPIの挙動において、開発者コミュニティから**「Lazy Coding（怠慢なコーディング）」**問題が多数報告されている。
   * 現象詳細: コードの修正を依頼した際、変更点以外を // ... existing code ... と省略して出力する傾向が強い。
   * 影響: Clineのような自動編集ツールは、AIが出力したコードでファイルを上書きするため、省略されると元のコードが消失してファイルが壊れる。これを防ぐためのプロンプト調整に手間取る。
 * コスト: キャッシュ機能（Prompt Caching）はあるものの、適用条件が厳しく、Anthropicに比べて割引率の恩恵を受けにくい。
 * 判定: コーディング用途の信頼性が低下しているため不採用。
2.2 Google "Gemini 1.5 Pro" の評価と棄却理由
 * 性能: コンテキストウィンドウ（記憶量）が巨大で、本一冊分以上のコードを読める。
 * コスト: 非常に安価。コストパフォーマンスだけなら最強。
 * 欠点: **「Instruction Following（指示追従性）」**の甘さ。
   * 現象詳細: 「C#のこの記法を使って」と指定しても、学習データに含まれる一般的な（古い）書き方に戻ってしまうことや、ファイル出力のフォーマットを微妙に間違えてClineがエラーを吐く頻度が他社より高い。
 * 判定: 「思考の整理」には良いが、「正確なファイル編集」においてストレス要因となるため、今回は**次点（保留）**とする。
2.3 DeepSeek "DeepSeek V3" の評価と棄却理由
 * 性能: 2024年12月末にリリースされ、GPT-4oに匹敵する性能を叩き出した話題のモデル。
 * コスト: 破壊的な安さ（他社の1/10〜1/20）。
 * 致命的欠陥: 地政学的リスク（チャイナリスク）。
   * 詳細: 運営元が中国企業であり、中国のサイバーセキュリティ法の影響下にある。API経由のデータといえども、サーバー内のログがどのように当局に扱われるか不透明。
   * ユーザーの「セキュリティが心配すぎる」という強い意向に対し、リスクを排除しきれない。
 * 判定: セキュリティ要件を満たさないため完全除外。
2.4 Anthropic "Claude 3.5 Sonnet" の評価と採用理由
 * 性能: 2024年12月現在、**「コーディング能力において世界一」**との評価がエンジニア界隈で定着している。
   * 特長: 複雑なロジックの理解力が高く、GPT-4oのような「省略」をせず、指示通りにフルコードを書く実直さがある。
 * コスト技術（重要）: **「Prompt Caching（プロンプトキャッシュ）」**機能が決定打となった。
   * 仕組み: コーディング作業では「同じソースコードファイル」を何度もAIに送る。Claudeはこの「繰り返し送られる部分」をサーバー側で一時保存し、2回目以降の読み込み料金を90%割引する。
   * 影響: これにより、本来なら月1万円かかるような開発作業が、月1,000円〜2,000円程度に圧縮可能となった。
 * セキュリティ: 米国企業（サンフランシスコ）。AWSやGoogleが出資しており、エンタープライズ（企業）向けのセキュリティ基準（SOC2等）に準拠。APIデータは学習利用されない。
 * 判定: 性能、コスト（キャッシュ込み）、セキュリティの全ての要件を満たすため採用。

第4章：セキュリティ仕様の詳細
ユーザーの不安要素である「セキュリティ」が技術的にどう担保されるか解説する。
4.1 データフローの透明性
 * 構成: VS Code (ローカル) → [暗号化通信 HTTPS] → Anthropic API Server (米国)
 * 学習利用: Anthropicの商用利用規約（Commercial Terms）において、「API経由で送信されたCustomer Contentは、当社のモデルトレーニングには使用しない」と明記されている。これは無料のChatGPT（Web版）とは明確に異なる扱いである。
4.2 ローカルファイルへのアクセス権限
 * 拡張機能「Cline」は、あくまでユーザーのPC上で動作するプログラムである。
 * AI（クラウド上のClaude）が勝手にPC内のファイルを覗くことはできない。Clineが「このファイルを読みますか？」とユーザーに提案し、ユーザーがそれを許可した場合にのみ、そのファイルの中身がテキストデータとしてAPIに送信される。
 * ファイルの書き換えやコマンド実行（例: npm install）も同様に、全て**Human-in-the-loop（人間の承認プロセス）**を経由するため、AIの暴走によるデータ消失リスクは防がれている。
