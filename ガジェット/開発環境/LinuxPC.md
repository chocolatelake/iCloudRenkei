# 🛒 Radxa X4 構成リスト（改造・超小型ルート）

**コンセプト:** クレジットカードサイズの基板で作る、iPad合体型の最強モバイルLinuxマシン。
**予算目安:** 2.5万円 〜 3万円

## 📦 必須購入パーツ

### 1. 本体・基板関連
- [ ] **Radxa X4 本体**
    - **必須条件:** CPU Intel N100 / **メモリ 16GB**
    - ※注意: メモリは後から増設不可。絶対に16GB版を買うこと。
- [ ] **M.2 SSD (2230サイズ)**
    - 容量: 512GB推奨
    - ※注意: 普通のPC用（2280）ではなく、短い「2230」サイズ。
    - 推奨: KIOXIA BG4, Western Digital SN740 など
- [ ] **Radxa X4専用 ヒートシンクファン**
    - ※汎用品は固定が難しいため、専用品を推奨。
- [ ] **WiFi / Bluetooth モジュール + アンテナ**
    - ※本体に内蔵されていない場合があるため要確認。「アンテナ付き」が必須。
    - 推奨: Radxa Wireless Module A8 または Intel AX210

### 2. iPad接続・映像関連
- [ ] **HDMI キャプチャボード (USB-C接続)**
    - UVC対応（Amazonで2,000円前後の安いものでOK）
- [ ] **Micro HDMI - HDMI ケーブル**
    - ※Radxa側が「Micro HDMI」端子のため、変換または専用ケーブルが必要。

### 3. 電源関連
- [ ] **モバイルバッテリー (PD 30W以上出力)**
    - ※スマホ用の18W充電器などでは起動しない可能性あり。
- [ ] **USB-C to C ケーブル**
    - PD対応（60W対応など）のしっかりしたケーブル。

## 🛠 購入推奨サイト
* **Arace Tech** (公式代理店・在庫確実)
* **AliExpress** (Radxa Official Store)
* **千石電商 / 秋月電子** (国内在庫があれば最速)

## 📝 備考・セットアップ
1. パーツを組み立てる（SSD、WiFi、ファンを取り付け）。
2. USB-Cで給電し、Micro HDMIでiPad（キャプチャ経由）に接続。
3. キーボードをUSBまたはBluetoothで接続。
4. OS（Ubuntu Desktopなど）をインストールして完了。


# 🛒 Mini PC 構成リスト（完成品・安定ルート）

**コンセプト:** 買ったその瞬間から開発スタート。安定動作とコスパ重視のN100マシン。
**予算目安:** 2.5万円 〜 2.8万円

## 📦 必須購入パーツ

### 1. PC本体
- [ ] **Mini PC (Intel N100搭載機)**
    - **推奨機種:** GMKtec NucBox G3 / CHUWI LarkBox X / Beelink S12 Pro
    - **必須スペック:**
        - CPU: Intel N100
        - メモリ: **16GB**
        - ストレージ: 512GB SSD (最初から入っていることが多い)

### 2. 電源ハック（モバイル化）
- [ ] **USB PD トリガーケーブル (12V)**
    - **コネクタ形状:** 外径5.5mm / 内径2.5mm (機種により要確認だが大体これ)
    - **電圧:** **12V対応** と書かれているもの
- [ ] **モバイルバッテリー (PD 65W出力推奨)**
    - 45Wでも動くが、65Wあると高負荷時も安心。
    - **重要:** 仕様欄に「Output: 12V=3A」の記載があるか確認すること。

### 3. iPad接続・映像関連
- [ ] **HDMI キャプチャボード (USB-C接続)**
    - UVC対応（Radxa用と同じものでOK）
- [ ] **HDMI - HDMI ケーブル**
    - 普通のHDMIケーブル（両端が標準サイズ）。短めのほうがカフェで邪魔にならない。

## 🛠 購入推奨サイト
* **Amazon** (セール時が狙い目、クーポン適用を忘れないこと)

## 📝 備考・セットアップ
1. PC本体にトリガーケーブルを刺し、モバイルバッテリーに繋ぐ。
2. HDMIケーブルでiPad（キャプチャ経由）に接続。
3. 電源ボタンONでWindowsが起動する。
4. そのままWSLを使うか、Ubuntuをクリーンインストールして開発環境化する。
